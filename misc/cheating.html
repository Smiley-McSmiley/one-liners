CHEATING

While you look through my graphs, you may find yourself thinking:
"Should we really be using this technique?"
"Surely this is cheating?"
"I'm hungry."
All of these are normal thoughts, but the latter two are the ones this page will attempt to address.
(If you are hungry, I cannot be of assistance.)

MIN/MAX

(Insert link to min/max explanation page)
Is using the min() and max() functions cheating?
Here are the facts:
-min() and max() are built-in Desmos functions.
-2-argument min() and max() can be expressed as (x + y +- |x - y|)/2.
-n-argument min() and max() can be expressed in terms of min() and max().
Personally, I see no issue with using min() and max(). It can be implemented using only the basic operations and saves a lot of time.

LISTS

(Insert link to list explanation page)
Is using lists cheating?
Here are the facts:
-Lists are a feature of Desmos not normally present in mathematics.
-They allow for multiple equations to be drawn in the same line, but in a particularly inelegant way.
-There are elegant ways of using lists.
Personally, I don't like to use lists just for combining. However, I do use lists when they are a good abstraction.

POWER TOWERS

(Insert link to 0^x explanation page)
Is using 0^x cheating?
Here are the facts:
-0, infinity exception cases are a built-in feature of Desmos not normally present in mathematics.
-0^0, inf^inf, 0^inf etc. are not normally defined well outside of Desmos.
-Power towers can provide a great shortening to the character count in a line.
Personally, I used to dislike these, since I did not have an intuition for them.
However, with time and practice I have come to use them as I have every other technique.
